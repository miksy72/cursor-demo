# 테트리스

HTML, CSS, JavaScript만으로 만든 브라우저 테트리스 게임입니다.  
빌드 도구나 외부 라이브러리 없이 바로 실행할 수 있습니다.

## 프로젝트 소개

표준 10×20 보드에서 7종 테트로미노(I, O, T, S, Z, J, L)를 조작하는 클래식 테트리스입니다.  
렌더링·게임 상태·입력 처리를 분리해 단계적으로 기능을 쌓아 올린 **입문용 프론트엔드 프로젝트**입니다.

| 파일 | 설명 |
|------|------|
| `index.html` | 게임 보드, 점수, 버튼, 조작법 안내 |
| `style.css` | 레이아웃, 보드 그리드, 블록 색상 |
| `script.js` | 게임 로직, 렌더링, 키보드 입력 |

## 실행 방법

### 1. 파일 탐색기에서 열기

1. `tetris` 폴더로 이동합니다.
2. `index.html`을 더블 클릭하거나 브라우저로 끌어다 놓습니다.

### 2. VS Code / Cursor Live Server (선택)

`index.html`에서 **Open with Live Server**를 선택합니다.

### 3. 로컬 HTTP 서버 (선택)

```bash
cd tetris
python -m http.server 8080
```

브라우저에서 `http://localhost:8080` 을 엽니다.

### 4. GitHub Pages (배포 후)

저장소 루트를 Pages 소스로 설정한 경우:

```
https://<사용자명>.github.io/cursor-demo/tetris/
```

## 조작법

**시작** 또는 **재시작** 후 보드에 포커스가 있어야 키 입력이 동작합니다.  
입력이 안 되면 보드를 한 번 클릭한 뒤 다시 시도하세요.

| 키 | 동작 |
|----|------|
| `←` (ArrowLeft) | 왼쪽으로 한 칸 이동 |
| `→` (ArrowRight) | 오른쪽으로 한 칸 이동 |
| `↓` (ArrowDown) | 한 칸 빠르게 내리기 (충돌 시 이동하지 않음) |
| `↑` (ArrowUp) | 시계 방향 90° 회전 (충돌 시 회전 취소) |
| `Space` | 즉시 낙하 (hard drop) |

| 버튼 | 동작 |
|------|------|
| **시작** | 자동 낙하 시작 |
| **재시작** | 보드·점수·상태 초기화 후 즉시 게임 시작 |

- 벽·고정 블록과 충돌하는 이동·회전은 적용되지 않습니다.

## 구현 기능

- 10×20 CSS Grid 보드 렌더링
- 7종 테트로미노 정의 및 무작위 생성
- `createPiece()`, `drawPiece()`, `renderBoard()` 분리
- `canMove()` 기반 충돌 판정 (벽·고정 블록·보드 밖 이동 차단)
- 자동 낙하 (`setInterval`, 중복 타이머 방지)
- 키보드 이동·회전·소프트 드롭·하드 드롭
- 가득 찬 줄 삭제 및 위 줄 낙하
- 줄 수에 따른 점수 계산
- 스폰 불가 시 게임 오버 표시
- 재시작 시 보드·점수·타이머·게임 상태 초기화

### 점수 규칙

블록 고정 직후, **한 번에** 제거된 줄 수에 따라 점수가 올라갑니다.

| 제거 줄 수 | 점수 |
|-----------|------|
| 1줄 | 100 |
| 2줄 | 300 |
| 3줄 | 500 |
| 4줄 | 800 |

### 게임 오버 조건

새 블록을 스폰 위치(보드 상단 중앙)에 둘 수 없으면 게임이 종료됩니다.  
**게임 오버** 문구가 표시되고 낙하·조작이 멈춥니다.

## 품질 점검 방법

### 1. 정적 검사

```bash
node --check tetris/script.js
```

문법 오류가 없으면 종료 코드 `0`입니다.

### 2. 브라우저 수동 점검

| 항목 | 확인 방법 | 기대 결과 |
|------|-----------|-----------|
| 리소스 로드 | DevTools → Network | `style.css`, `script.js`가 200으로 로드 |
| 콘솔 | DevTools → Console | 빨간 에러 없음 |
| 보드 | 페이지 로드 | 10×20 그리드 표시 |
| 낙하 | **시작** 클릭 | 블록이 주기적으로 아래로 이동 |
| 이동 | `←` `→` | 벽에서 멈춤 |
| 회전 | `↑` | 충돌 시 회전 취소 |
| 줄 삭제 | 가로줄 완성 | 줄 제거·점수 증가 |
| 게임 오버 | 보드 상단까지 채움 | 문구 표시·입력 정지 |
| 재시작 | **재시작** 클릭 | 초기화 후 즉시 낙하 재개 |
| 타이머 | **시작** 연속 클릭 | 낙하 속도가 빨라지지 않음 |

### 3. GitHub Pages 배포 후 점검

배포 URL(`…/tetris/`)에서 위 항목을 다시 확인합니다.  
특히 CSS·JS가 404가 아닌지 Network 탭에서 확인하세요.

## GitHub Pages 배포 방법

이 게임은 저장소 **루트의 `tetris/` 하위 폴더**에 있습니다.

### 사전 조건

- `tetris/` 폴더가 Git에 커밋·푸시되어 있어야 합니다.
- GitHub Pages는 기본적으로 `/`(루트) 또는 `/docs`만 소스로 선택할 수 있습니다.  
  → 루트를 소스로 쓰면 게임 URL은 **`/tetris/`** 경로가 됩니다.

### 배포 절차

1. 아래 파일을 커밋하고 `main` 브랜치에 푸시합니다.
   - `tetris/index.html`
   - `tetris/style.css`
   - `tetris/script.js`
   - `tetris/README.md`
2. GitHub 저장소 → **Settings** → **Pages**
3. **Build and deployment** → Source: **Deploy from a branch**
4. Branch: `main`, Folder: **`/ (root)`** → **Save**
5. 1~2분 후 `https://miksy72.github.io/cursor-demo/tetris/` 에 접속합니다.

### 배포 URL 안내 (선택)

저장소 루트(`https://…/cursor-demo/`)에는 `index.html`이 없으므로,  
방문자가 바로 게임에 도달하려면 루트 `README.md`에 위 링크를 안내하거나,  
루트에 `index.html` 리다이렉트 페이지를 추가할 수 있습니다.

### 주의사항

- `index.html`의 `style.css`, `script.js`는 **상대 경로**이므로 `/tetris/` URL에서 정상 동작합니다.
- 루트(`/`)만 Pages 소스로 설정한 채 `tetris/`를 `/docs`로 옮기지 않았다면,  
  `https://…/cursor-demo/` 직접 접속만으로는 게임이 열리지 않습니다.
